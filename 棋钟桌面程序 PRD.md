# 棋钟桌面程序 PRD

> 版本 1.0 | 状态：待开发 | 技术栈：C++17 + FLTK 1.4.x (静态编译)

------

## 1. 产品愿景与目标

打造一款 **二进制小于 800KB、内存低于 8MB、CPU 占用 ≈0%** 的跨平台棋钟工具，为棋类爱好者及比赛场景提供精准、无干扰、极速启动的计时体验。它必须：

- 无安装、无依赖，即点即用，删除即卸载。
- 界面清晰克制，所有操作均可在 1 次点击/快捷键内完成。
- 具备正式比赛所需的多种计时模式，同时保持代码和资源的极致精简。

------

## 2. 目标用户与使用场景

- **棋手日常练习**：本地或在线对局旁侧挂一个独立计时器，快捷键切换，不占用棋盘空间。
- **俱乐部/小型比赛**：放置于屏幕角落，窗口置顶，超大数字一目了然，不干扰对局。
- **触屏设备**：支持全屏无边框大按钮，触摸点击任意半区即切换计时方。
- **极客用户**：追求最小二进制，对 Electron/框架类软件零容忍，希望在多平台获得一致轻量体验。

## 3. 功能需求（Functional Requirements）

### 3.1 核心计时逻辑

| ID   | 需求描述                                                     | 优先级 |
| :--- | :----------------------------------------------------------- | :----- |
| F1   | 提供两个独立倒计时器（白方/黑方），最小显示单位为 0.1 秒（最后 10 秒内），其余时间精度为 1 秒。 | P0     |
| F2   | 计时器状态机：IDLE（初始/重置后）、RUNNING（某一方计时中）、PAUSED（双方暂停）、TIMEOUT（一方超时）。 | P0     |
| F3   | 单击左半区启动/切换至黑方计时（白方暂停），单击右半区启动/切换至白方计时（黑方暂停）；需视觉明确指示当前运行方。 | P0     |
| F4   | 支持键盘快捷键：**空格键**（切换计时方，首次启动计时）、**P 键**（暂停/继续）、**R 键**（重置到初始时间）。 | P0     |
| F5   | 超时处理：超时方时间显示 `0:00` 并闪烁，触发声音报警，同时自动暂停，需手动重置恢复。 | P0     |

### 3.2 时间控制模式

| ID   | 需求                                                         | 优先级 |
| :--- | :----------------------------------------------------------- | :----- |
| F6   | **绝对限时（Sudden Death）**：双方各 N 分钟，无补偿，先归零者负。 | P0     |
| F7   | **加秒制（Fischer）**：每次切换计时前，自动为当前运行方增加 S 秒（如 3+2：初始 3 分钟，每次加 2 秒）。 | P0     |
| F8   | **延迟制（Bronstein）**：切换后延迟 D 秒再开始扣减（若思考时间少于 D 秒，则不扣时间直接切换）。 | P1     |
| F9   | 允许用户分别设置主时间（时、分、秒）及增量/延迟秒数，范围：主时间 0~999 分钟，增量 0~99 秒。 | P0     |

### 3.3 预设与快速设置

| ID   | 需求                                                         | 优先级 |
| :--- | :----------------------------------------------------------- | :----- |
| F10  | 提供一组常用预设组合：`1+0`、`3+0`、`5+0`、`3+2`、`5+3`、`10+0`，选择后自动填表并应用。 | P0     |
| F11  | 预设列表旁有“自定义”选项，激活手动输入界面。                 | P1     |
| F12  | 当前使用的时间控制参数在标题栏或窗口一角展示（如“5 min + 3 sec”）。 | P2     |

### 3.4 设置持久化

| ID   | 需求                                                         | 优先级 |
| :--- | :----------------------------------------------------------- | :----- |
| F13  | 程序关闭时自动保存：当前时间控制模式、主时间、增量/延迟值、窗口位置、置顶状态。 | P0     |
| F14  | 使用纯文本键值对配置文件 `clock.conf`，位于可执行文件同级目录；若文件不存在或无效，则使用默认值（5+0 绝对限时）。 | P0     |

### 3.5 窗口与界面行为

| ID   | 需求                                                         | 优先级 |
| :--- | :----------------------------------------------------------- | :----- |
| F15  | **窗口置顶**：菜单或按钮切换，置顶时窗口始终在前。           | P1     |
| F16  | **无边框模式**：可切换为无标题栏窗口，仅显示双色计时区域 + 半透明小按钮，适合触屏或嵌入场景。 | P2     |
| F17  | 窗口大小固定为 600×350 像素（默认），不允许拉伸；数字根据窗口按比例缩放，确保清晰。 | P1     |
| F18  | 界面支持深色/浅色主题（至少深色默认），字体使用 FLTK 内置矢量字体避免外部依赖。 | P2     |

### 3.6 声音与通知

| ID   | 需求                                                         | 优先级 |
| :--- | :----------------------------------------------------------- | :----- |
| F19  | 超时发出系统蜂鸣（如 `MessageBeep` 或 `\a`），不依赖任何音频文件；可静音。 | P0     |
| F20  | 低电量式预警：当某一方剩余时间低于 10 秒时，数字变红并每秒发出短促提示音（可选）。 | P2     |

## 4. 非功能需求（Non-Functional Requirements）

### 4.1 资源占用（强制约束）

| 指标           | 目标值                                            | 验证方式                       |
| :------------- | :------------------------------------------------ | :----------------------------- |
| 可执行文件大小 | ≤ 800 KB（Windows 静态链接 /MT，strip，UPX 可选） | 资源管理器文件大小             |
| 私有工作集内存 | ≤ 8 MB（空闲状态计时）                            | 任务管理器 / htop              |
| CPU 占用       | 计时运行期间 ≤ 0.5%（现代 x86 处理器）            | 性能监视器，刷新间隔设为 100ms |
| 启动时间       | ≤ 0.2 秒（冷启动至窗口出现）                      | 秒表测量                       |
| 磁盘写入       | 仅保存配置时写入，频率 ≤ 1次/关闭                 | 无持续 I/O                     |

### 4.2 性能与精度

- 时间基准：`std::chrono::steady_clock`（单调时钟），刷新频率 20 Hz（每 50ms 更新一次显示），计算使用“绝对时间差”避免漂移。
- 切换计时方时，误差 ≤ 0.01 秒。

### 4.3 兼容性

- 操作系统：Windows 10/11（x64），Linux（Ubuntu 20.04+），macOS 11+（均为 Intel/ARM）。
- 编译链：GCC 9+/Clang 10+/MSVC 2019+，均使用静态链接 FLTK。
- 无需 GPU 加速，纯 CPU 软件渲染（FLTK 默认 X11/Win32 GDI 后端）。

### 4.4 可维护性

- 代码行数目标 ≤ 2000 行有效 C++ 代码，单文件主逻辑 + 若干小工具文件。
- 零外部依赖（除 C++17 标准库和静态 FLTK）。
- 配置解析自实现，不使用第三方 JSON/INI 库。

## 6. 技术架构要点

- **图形库**：FLTK 1.4.x（使用 `fltk-config --static --use-images` 编译），链接参数剥离所有未使用的 widget 以缩小体积。
- **核心类设计**：
  - `ChessClock`：继承 `Fl_Double_Window`，管理两方计时器。
  - `PlayerTimer`：封装剩余时间、计数方式、超时回调。
  - `Prefs`：配置读写静态方法。
- **计时实现**：使用 `Fl::add_timeout(0.05, timer_cb)` 驱动刷新（20fps），内部用 `std::chrono::steady_clock::now()` 计算真实流逝时间，每次回调中重绘。
- **事件处理**：重写 `handle(int event)` 捕获点击左右区域、键盘事件；窗口关闭事件保存配置。
- **声音**：跨平台蜂鸣通过条件编译实现：Windows 用 `MessageBeep(MB_ICONEXCLAMATION)`，其他平台用 `std::cout << '\a'`（或直接调用 `Beep` 在 POSIX 用 `putchar('\a')`）。

## 8. 风险与约束

- **FLTK 静态链接体积**：需手动裁剪不用的 widget，测试时必须确认链接器 tree-shaking 效果，否则可能超出目标体积。解决：只显式链接 `fltk`, `fltk_gl` 勿用，完全剔除图像加载、文件选择器等组件。
- **跨平台声音一致性**：`\a` 行为受终端控制，纯 GUI 程序可能不响。解决：Windows 用 `MessageBeep`，Linux 可尝试 `XBell` 或直接播放一段频率（若 ALSA 可用，但不应引入依赖，退而求其次用 `std::cout << '\a'` 并保持控制台隐藏）。
- **高 DPI 下数字清晰度**：FLTK 默认不支持缩放，需通过 `Fl::screen_scale()` 获取比例并手动选择字号，确保 4K 屏下文字不模糊（FLTK 1.4 已较好支持）。

## 9. 验收标准

- 在 Windows 10/11 上，二进制大小 ≤ 800 KB，运行内存（空闲计时）≤ 8 MB，CPU 始终 ≤ 0.5%。
- 模式切换、加秒、延迟逻辑经手动测试与实际棋规一致。
- 切换方计时误差累计 10 分钟不超过 ±0.1 秒。
- 配置保存后重启程序完全恢复上次设置。
- 无边框模式可正常拖拽移动（通过按住任意区域模拟拖拽）。
